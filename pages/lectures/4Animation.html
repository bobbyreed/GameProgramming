Why Not setInterval?

The Traditional Approach Falls Short

‚ùå setInterval Problems

  • Doesn't sync with browser refresh
  • Continues in background tabs
  • Can cause janky animations
  • Wastes battery on mobile
  • Frame skipping and stuttering

‚úÖ requestAnimationFrame Benefits

  • Synced to monitor refresh rate
  • Pauses in background tabs
  • Smooth 60 FPS when possible
  • Battery efficient
  • Browser optimized timing
Key Insight: The browser knows when it's the best time to draw!

Meet requestAnimationFrame

The modern way to create smooth animations:

// Basic animation loop structure function animate(timestamp) { // timestamp: DOMHighResTimeStamp in milliseconds // Do your animation work here updateAnimation(); drawFrame(); // Request the next frame requestAnimationFrame(animate); } // Start the animation requestAnimationFrame(animate);

Key Feature: Automatically provides a high-resolution timestamp!

This timestamp helps us calculate smooth, frame-rate independent animations.

Animation Gotchas to Avoid

‚ö†Ô∏è Common Mistakes

  • Forgetting to clear: Creates "painting" effect
  • Not using deltaTime: Speed varies with frame rate
  • Recursive setTimeout: Less efficient than rAF
  • Multiple animation loops: Use one main loop!
  • Heavy calculations in render: Do math in update()
Pro Tip: Always separate your update logic (physics, collisions) from your rendering logic (drawing). This makes your code cleaner and more performant!
