Our Project Structure

  • index.html // Game entry point
  • /js/ // Main JavaScript root
    • game.js // Primary game module
    • /core/ // Engine modules

    • /utils/ // Utility functions

Pro Tip: Keep your structure predictable - your future self will thank you!

The Module Pattern

// Basic module structure function MyModule() { // Private variables var secret = "Can't touch this!"; // Public interface return { publicMethod: function() { return "I'm accessible!"; } }; } // Export for use in other files module.exports = MyModule;

Key Concept: Modules help us encapsulate functionality and control what's public vs private!

Pro Tip: Dynamic Canvas Creation

// utils/utils.canvas.js module.exports = { generateCanvas: function(w, h) { var canvas = document.createElement('canvas'); canvas.width = w; canvas.height = h; // Handle high-DPI displays var ctx = canvas.getContext('2d'); var ratio = window.devicePixelRatio || 1; canvas.width = w * ratio; canvas.height = h * ratio; canvas.style.width = w + 'px'; canvas.style.height = h + 'px'; ctx.scale(ratio, ratio); return canvas; } };

This ensures crisp graphics on all devices!

‚ö†Ô∏è Common Canvas Pitfalls

1. Forgetting to clear the canvas

Always use ctx.clearRect(0, 0, width, height) before redrawing!

2. Wrong coordinate math

Remember: Y increases downward, not upward!

3. Not saving/restoring context

Use ctx.save() and ctx.restore() for transformations!
