What is Input Handling?

Input handling is how your game listens to and responds to player actions.

The Input Chain

1. Player presses key ‚Üí Browser fires event

2. Event listener catches it ‚Üí Your code responds

3. Game state updates ‚Üí Character moves!

Real World Examples:

  • WASD for movement in most PC games
  • Space bar for jumping
  • Mouse for aiming and shooting
  • Combo keys for special moves

Event Listeners: Your Game's Ears

Event listeners let your game "hear" when keys are pressed.

// Basic event listener document.addEventListener('keydown', function(event) { console.log('Key pressed:', event.keyCode); }); // Listening for key release document.addEventListener('keyup', function(event) { console.log('Key released:', event.keyCode); });
Pro Tip: Always pair keydown with keyup! This tells you when a key starts being pressed AND when it's released.

Switch Statements: Handling Multiple Keys

Use switch statements to elegantly handle different keys.

document.addEventListener('keydown', function(event) { switch(event.keyCode) { case 37: // Left arrow player.moveLeft(); break; case 39: // Right arrow player.moveRight(); break; case 38: // Up arrow player.jump(); break; case 32: // Space bar player.shoot(); break; } });

The Problem: Input Lag

‚ùå Event-Only Approach

  • Fires once per keypress
  • Has keyboard repeat delay
  • Can't handle multiple keys well
  • Feels sluggish in games

‚úÖ State Tracking

  • Tracks which keys are down
  • Checked every frame
  • Perfect for simultaneous inputs
  • Feels smooth and responsive

üí° The Solution: State Management

Track which keys are currently pressed in an object, then check that object in your game loop!

Key State Management Pattern

Create an object to track the state of each key.

// Create a keys object to track state var keys = { left: false, right: false, up: false, down: false }; // Update state when keys are pressed document.addEventListener('keydown', function(event) { if (event.keyCode === 37) keys.left = true; if (event.keyCode === 39) keys.right = true; }); // Update state when keys are released document.addEventListener('keyup', function(event) { if (event.keyCode === 37) keys.left = false; if (event.keyCode === 39) keys.right = false; });

Using Key States in Your Game Loop

function update() { // Check key states every frame if (keys.left) { player.x -= player.speed; } if (keys.right) { player.x += player.speed; } if (keys.up) { player.y -= player.speed; } if (keys.down) { player.y += player.speed; } // Continue game loop requestAnimationFrame(update); }
Why This Works: Every frame, we check the current state of all keys. Multiple keys pressed = multiple if statements execute = smooth diagonal movement!

Advanced Pattern: The keysDown Module

Create a reusable module with getters for clean syntax.

function keysDown() { this.isPressed = {}; var left, right, up, down; document.onkeydown = function(ev) { if (ev.keyCode === 37) left = true; if (ev.keyCode === 39) right = true; }; document.onkeyup = function(ev) { if (ev.keyCode === 37) left = false; if (ev.keyCode === 39) right = false; }; // Using getters for clean syntax Object.defineProperty(this.isPressed, 'left', { get: function() { return left; } }); return this; }

Event Bubbling & Capturing

Event Flow

Capturing Phase: Event travels down from window to target

Target Phase: Event reaches the actual element

Bubbling Phase: Event bubbles back up to window

// Third parameter controls capturing element.addEventListener('keydown', handler, true); // Capture element.addEventListener('keydown', handler, false); // Bubble (default)
For Games: Usually, you want the default (bubbling). Just use document.addEventListener for global input!

Preventing Default Browser Behavior

Stop browsers from doing their default actions (like scrolling).

document.addEventListener('keydown', function(event) { // Prevent arrow keys from scrolling page if (event.keyCode >= 37 && event.keyCode <= 40) { event.preventDefault(); } // Prevent spacebar from scrolling if (event.keyCode === 32) { event.preventDefault(); } });
Common Keys to Prevent:
  • Arrow keys (37-40) - Prevent scrolling
  • Spacebar (32) - Prevent page jump
  • Tab (9) - If using for game control

Input Handling Best Practices

‚úÖ Do This

  • Use state tracking for game controls
  • Always pair keydown with keyup
  • Prevent default for game keys
  • Organize input code in modules
  • Test with multiple key combos

‚ùå Avoid This

  • Relying only on keydown events
  • Forgetting to clear key states
  • Not preventing default actions
  • Hard-coding key codes everywhere
  • Ignoring keyboard repeat delays

Common Input Handling Pitfalls

‚ö†Ô∏è Watch Out For:

  • Stuck Keys: Key stays "pressed" if player tabs away while holding key
    • Solution: Listen to window blur event and clear all keys
  • Browser Shortcuts: Ctrl+W closes tab, Ctrl+T opens new tab
    • Solution: Use preventDefault for game keys
  • Keyboard Ghosting: Some key combos don't register on certain keyboards
    • Solution: Test multiple key combinations, offer alternatives

Fixing the Stuck Key Problem

// Clear all keys when window loses focus window.addEventListener('blur', function() { keys.left = false; keys.right = false; keys.up = false; keys.down = false; }); // Or use a cleaner loop approach window.addEventListener('blur', function() { for (var key in keys) { keys[key] = false; } });
Why This Matters: If a player holds a key and then Alt+Tabs, your game won't receive the keyup event, leaving the key "stuck" in the pressed state!

Putting It All Together

A complete, production-ready input handler.

var InputManager = { keys: {}, init: function() { document.addEventListener('keydown', this.onKeyDown.bind(this)); document.addEventListener('keyup', this.onKeyUp.bind(this)); window.addEventListener('blur', this.clearAll.bind(this)); }, onKeyDown: function(e) { this.keys[e.keyCode] = true; if (e.keyCode >= 37 && e.keyCode <= 40) e.preventDefault(); }, onKeyUp: function(e) { this.keys[e.keyCode] = false; }, clearAll: function() { this.keys = {}; }, isPressed: function(keyCode) { return this.keys[keyCode] === true; } };
