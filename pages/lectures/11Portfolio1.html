Understanding "Modding" 🔧

Modding = Modifying existing game systems to create something new

The Bengston Engine gives you:

  • ✅ A working game loop (update, render, input handling)
  • ✅ High-DPI canvas rendering system
  • ✅ Frame rate management and display
  • ✅ A movable player entity with keyboard controls
  • ✅ Modular, organized code structure

You will add:

  • 🎮 Your unique gameplay mechanics
  • 🎨 Custom visuals and aesthetics
  • 🎯 Goals, challenges, and win/lose conditions
  • ⚡ Special effects and polish

The Bengston Engine Core Systems

1. Player System (players/player.js)

Movement, boundaries, controls, visual representation

Mod Ideas: Change speed, add acceleration, different movement patterns, animations

2. Rendering System (core/game.render.js)

What appears on screen, how it looks, visual effects

Mod Ideas: Background colors, shapes, images, particles, UI elements

3. Update System (core/game.update.js)

Game logic, entity updates, collision detection, game state

Mod Ideas: Add enemies, collectibles, scoring, win/lose conditions

Additional Systems to Consider

4. Input System (core/game.keybindings.js)

How player interacts with the game

Mod Ideas: Add shooting (spacebar), dash (shift), special abilities

5. World System (world/)

Level design, boundaries, multiple rooms

Mod Ideas: Obstacles, different levels, procedural generation

6. New Entity Types

Objects beyond the player

Mod Ideas: Enemies, projectiles, collectibles, hazards, power-ups

Exercise 3: Your First Mod 🛠️

⏱️ 15 Minutes - Make Three Changes

Change 1: Customize the Player

In players/player.js, modify:

  • Player size, color, or shape
  • Movement speed
  • Starting position

Change 2: Modify the Viewport

In game.js or core/game.render.js:

  • Canvas size
  • Background color
  • Add some custom text or shapes

Change 3: Adjust Game Feel

In core/game.loop.js:

  • Change target FPS
  • Toggle FPS display on/off

Quick Reference: Where to Find Things

js/ ├── game.js // Main game initialization ├── core/ │ ├── game.loop.js // Game loop & FPS │ ├── game.render.js // Rendering logic │ └── game.update.js // Update logic ├── players/ │ └── player.js // Player entity ├── utils/ │ ├── utils.canvas.js // Canvas utilities │ └── utils.misc.js // Helper functions └── world/ └── [levels] // Level definitions
Workflow Tip: Always test after EACH small change. Don't make 10 changes at once!

Common First Modifications

Easy Wins (Start Here!)

  • 🎨 Change colors and visual appearance
  • 📏 Adjust sizes and dimensions
  • ⚡ Modify speeds and timing
  • 🎯 Change starting conditions
  • 📝 Add custom UI text or displays

Intermediate Challenges

  • ➕ Add new keybindings for actions
  • 🎮 Create new entity types (based on player)
  • 💥 Implement basic collision detection
  • 🔢 Add score or health systems
  • 🎲 Introduce randomness or procedural elements

Best Practices for Your Project

✅ Do This:

  • Commit to Git frequently with clear messages
  • Test your game often while developing
  • Comment your code, especially NEW code
  • Keep a TODO list or task tracker
  • Start simple, add complexity gradually
  • Ask for help when stuck for >30 minutes

❌ Avoid This:

  • Making massive changes without testing
  • Trying to implement everything at once
  • Copying code you don't understand
  • Ignoring errors or console warnings
  • Waiting until the last minute
